Чтобы добавить новую запись в менеджер загрузки UEFI с помощью утилиты

`efibootmgr` в Linux, используйте команду с набором необходимых параметров. 

Синтаксис команды

Основной синтаксис выглядит следующим образом:

bash

```
sudo efibootmgr -c -d /dev/sdX -p Y -L "Название_Записи" -l "\\EFI\\путь\\к\\загрузчику.efi"
```

Use code with caution.

Объяснение параметров

- `-c` (или `--create`): Указывает утилите создать новую загрузочную запись.
- `-d /dev/sdX` (или `--disk`): Определяет диск, на котором находится системный раздел EFI (ESP). Замените `sdX` на соответствующее имя вашего диска, например, `sda` или `nvme0n1`.
- `-p Y` (или `--part`): Указывает номер раздела ESP на указанном диске. Замените `Y` на номер раздела, например, `1` или `2`. Как правило, ESP является первым или вторым разделом и отформатирован в FAT32.
- `-L "Название_Записи"` (или `--label`): Задает отображаемое имя для новой записи в меню загрузки UEFI. Используйте понятное название, например, "Arch Linux" или "Windows Boot Manager".
- `-l "\\EFI\\путь\\к\\загрузчику.efi"` (или `--loader`): Указывает путь к исполняемому файлу загрузчика EFI (обычно с расширением `.efi`) относительно корня ESP. 

**Важно:** Для указания пути в параметре `-l` **необходимо использовать обратную косую черту** (`\`) в качестве разделителя каталогов, как в Windows, а не прямую косую черту (`/`), используемую в Linux. 

Пример использования

Предположим, ваш системный раздел EFI находится на `/dev/sda1`, и вы хотите добавить запись для загрузчика GRUB с путем `\EFI\GRUB\grubx64.efi` и названием "Gentoo":

bash

```
sudo efibootmgr -c -d /dev/sda -p 1 -L "Gentoo" -l "\\EFI\\GRUB\\grubx64.efi"
```

Use code with caution.

Дополнительные шаги

1. **Проверьте существующие записи:** Перед добавлением или после него можно просмотреть текущий список записей и их идентификаторы, выполнив команду `sudo efibootmgr`.
2. **Измените порядок загрузки:** Если новая запись не становится первой в списке загрузки, вы можете изменить приоритет с помощью опции `-o`, указав нужный порядок идентификаторов (например, `sudo efibootmgr -o 0001,0000`).
